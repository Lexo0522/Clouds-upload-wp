# 批量迁移工具使用指南

## 功能概述

批量迁移工具可以将现有的本地附件批量迁移到云存储，支持：

- ✅ 一键迁移所有本地附件
- ✅ 增量迁移（只迁移未上传的文件）
- ✅ 迁移进度实时显示
- ✅ 失败重试机制
- ✅ 详细迁移日志
- ✅ 支持所有云服务商

## 访问迁移工具

进入 WordPress 后台 → 设置 → 云存储迁移

## 使用步骤

### 1. 查看统计信息

打开迁移页面后，会自动显示当前网站的附件统计：

- **总附件数**：网站所有媒体附件的总数
- **已上传**：已上传到云端的附件数
- **未上传**：尚未上传的附件数
- **失败**：之前上传失败的附件数
- **总大小**：附件文件总大小（估算）

### 2. 选择迁移选项

#### 增量迁移（推荐）

勾选「增量迁移」，系统会自动跳过已上传到云端的附件，只迁移未上传的文件。

**优点**：
- 节省时间
- 避免重复上传
- 节省流量和费用

**适用场景**：
- 首次迁移
- 新增附件后补充迁移
- 之前的迁移中断后继续

#### 重试失败的迁移

勾选「重试失败的迁移」，系统会只重试之前迁移失败的附件。

**适用场景**：
- 修复网络问题后重试
- 修复配置错误后重试
- 部分附件首次上传失败

**注意**：这两个选项是互斥的，只能选择其中一个。

### 3. 开始迁移

点击「开始迁移」按钮，系统会开始处理：

1. 获取需要迁移的附件列表
2. 分批处理（每批 5 个附件）
3. 上传文件到云端
4. 更新附件元数据
5. 处理缩略图
6. 显示进度和日志

### 4. 监控进度

迁移过程中，可以实时查看：

#### 进度条
- 显示百分比
- 动态更新

#### 详细信息
- 总计：需要迁移的附件总数
- 已处理：已处理的附件数
- 成功：成功上传的数量
- 失败：上传失败的数量
- 状态：当前运行状态

#### 迁移日志
- 实时显示操作日志
- 不同颜色区分状态
- 绿色：成功
- 红色：失败
- 蓝色：信息

### 5. 取消迁移

如需中断迁移，点击「取消迁移」按钮：

- 系统会在当前批次完成后停止
- 已上传的文件会保留
- 可以稍后继续增量迁移

### 6. 重试失败项

如果迁移完成后有失败的附件：

1. 查看失败详情列表
2. 修复问题（如网络、配置等）
3. 点击「重试失败项」按钮
4. 系统会只重新处理失败的附件

## 迁移策略建议

### 首次迁移

```
建议使用增量迁移
估算时间 = 附件数量 × 平均文件大小 ÷ 上传速度
```

**建议**：
- 在网站访问量低时执行
- 分时段进行（如每天迁移一部分）
- 确保云存储配置正确

### 大型网站

对于有大量附件的网站：

1. **分批迁移**
   - 先迁移最近的附件
   - 再迁移历史附件

2. **错峰迁移**
   - 选择凌晨时段
   - 避开访问高峰

3. **监控资源**
   - 关注服务器负载
   - 监控内存使用
   - 检查 PHP 超时设置

### 失败处理

#### 常见失败原因

1. **网络问题**
   - 检查服务器网络连接
   - 检查防火墙设置
   - 验证云服务商连接

2. **文件权限**
   - 检查文件是否可读
   - 检查目录权限
   - 验证文件是否存在

3. **配置错误**
   - 验证云存储配置
   - 检查密钥是否正确
   - 确认 Bucket 设置

4. **资源限制**
   - PHP 内存不足
   - 执行时间超时
   - 上传大小限制

#### 解决方案

1. **修复问题**
   - 根据错误日志定位问题
   - 修复配置或环境

2. **重试失败项**
   - 点击重试按钮
   - 系统自动处理失败的附件

3. **手动处理**
   - 在失败列表中查看附件 ID
   - 手动检查文件
   - 单独重新上传

## 性能优化

### 服务器端

**PHP 配置**：
```ini
memory_limit = 256M
max_execution_time = 300
upload_max_filesize = 100M
post_max_size = 100M
```

**数据库优化**：
- 确保 `postmeta` 表有索引
- 考虑使用对象缓存

### 客户端

**浏览器建议**：
- 使用现代浏览器（Chrome、Firefox、Edge）
- 保持浏览器窗口开启
- 避免同时进行其他操作

## 技术细节

### 分批处理

- 每批处理 5 个附件
- 批次间隔 500ms
- 避免服务器过载
- 提供实时反馈

### 断点续传

- 迁移状态保存在数据库
- 支持中断后继续
- 自动跳过已完成项

### 缩略图处理

- 自动上传所有尺寸缩略图
- 保留元数据信息
- 支持 WordPress 标准图片尺寸

### 错误处理

- 单个文件失败不影响其他文件
- 详细错误日志记录
- 支持单独重试

## AJAX 接口

### 获取统计信息
```
Action: wpmcs_get_migration_stats
响应: {
    total_attachments: 数量,
    uploaded_attachments: 数量,
    not_uploaded: 数量,
    failed_attachments: 数量,
    total_size: 大小
}
```

### 开始迁移
```
Action: wpmcs_start_migration
参数: {
    incremental: true/false,
    retry_failed: true/false
}
```

### 处理批次
```
Action: wpmcs_process_migration_batch
参数: {
    batch_size: 数量
}
```

### 获取进度
```
Action: wpmcs_get_migration_progress
响应: {
    status: 状态,
    total: 总数,
    processed: 已处理,
    success: 成功数,
    failed: 失败数
}
```

### 重试失败项
```
Action: wpmcs_retry_failed
```

### 取消迁移
```
Action: wpmcs_cancel_migration
```

## 常见问题

### Q: 迁移会删除本地文件吗？
A: 不会。迁移只是上传文件到云端，本地文件会保留。如果需要清理本地文件，请在迁移成功后手动处理。

### Q: 迁移过程中可以上传新文件吗？
A: 可以。新上传的文件会自动使用云端存储，不会影响正在进行的迁移任务。

### Q: 迁移失败会影响网站吗？
A: 不会。迁移失败的文件会跳过，网站继续使用本地文件。可以稍后重试。

### Q: 可以只迁移部分文件吗？
A: 目前不支持选择特定文件迁移。可以通过增量迁移来跳过已上传的文件。

### Q: 迁移速度慢怎么办？
A: 速度取决于：
- 服务器与云服务商的网络
- 文件大小和数量
- 服务器性能
建议分时段迁移，或使用更接近云服务商节点的服务器。

### Q: 迁移中断后如何继续？
A: 直接点击「开始迁移」，勾选「增量迁移」，系统会自动跳过已上传的文件。

### Q: 如何确认迁移成功？
A: 
1. 查看迁移日志，确认没有错误
2. 检查媒体库，查看云端状态列
3. 访问前台页面，确认图片显示正常
4. 检查图片 URL 是否为云端域名
