# 云端自动上传说明

本文档说明 WP Multi Cloud Storage 在开启后，图片是否会自动上传到你配置的云储存，以及前台 URL 替换、历史内容迁移和常见问题。

## 结论

- 新上传的图片会自动上传到你当前配置的云储存。
- 是否把前台/文章里显示的图片地址切换到云端，由 `replace_url` 控制。
- 已经存在的旧图片，不会因为你刚启用插件就自动重写文章内容，需要执行迁移或重写流程。

## 自动上传的工作方式

上传链路分两层：

1. WordPress 完成本地上传后，插件会拦截上传流程。
2. 插件会把图片同步上传到你配置的云储存，并保存云端地址和附件元数据。

相关实现位置：

- `includes/class-wpmcs-upload-interceptor.php`
- `includes/class-wpmcs-upload-manager.php`
- `wp-multi-cloud-storage.php`

### 立即上传还是队列上传

- 如果 `async_upload` 关闭，图片会在上传流程中直接同步到云端。
- 如果 `async_upload` 开启，图片会先进入异步队列，随后再上传到云端。

无论哪种方式，目标都是云端存储，不是只保存在本地。

## 前台地址是否会变

图片上传到云端后，前台显示的地址是否切换，取决于 `replace_url`：

- `replace_url = 1`：附件 URL、图片 `src`、`srcset` 等会尽量切换到云端地址。
- `replace_url = 0`：文件仍会上传到云端，但前台可能继续显示本地地址。

这意味着：

- “上传到云端” 和 “前台显示云端地址” 是两个独立开关。
- 如果你只关心备份和对象存储，自动上传就够了。
- 如果你希望站点前台直接使用腾讯云地址，需要开启 `replace_url`。

## 腾讯云 COS 必要配置

腾讯云 COS 常用字段如下：

- `Secret ID`
- `Secret Key`
- `Bucket`
- `Region`
- `Domain`（可选，但建议配置，用于自定义访问域名）

如果你只是想验证“是否会上传到云端”，这四项至少要正确配置：

- `Secret ID`
- `Secret Key`
- `Bucket`
- `Region`

## 新图和旧图的区别

### 新上传图片

只要插件已启用且配置完整，新上传的图片会自动进入云端。

### 旧图片

旧图片不会因为插件启用而自动“改写文章内容”。

如果文章正文里已经写死了旧的本地图片地址，通常需要：

- 重新执行迁移
- 或执行一次内容重写

否则，云端里可能已经有文件，但文章仍然引用旧链接。

## 你可以如何判断是否成功

判断顺序建议如下：

1. 看测试连接是否通过。
2. 上传一张新图片。
3. 检查云储存控制台是否出现对应对象。
4. 检查文章里图片地址是否变成云端域名。

如果上传成功但“文件访问测试”返回 `403`，不一定代表上传失败，通常是以下原因之一：

- 桶是私有读
- 自定义域名未开放匿名访问
- 防盗链或权限策略阻止了匿名 `GET`

这种情况下，文件已经上传成功，但测试连接的“匿名访问验证”会失败。

## 常见问题

### Q: 为什么上传后本地看得到，但云端访问不了？

A: 先确认：

- 你的桶权限
- 自定义域名配置
- 是否存在防盗链

### Q: 为什么旧文章里的图片地址没有变？

A: 因为旧内容不会自动批量重写。需要执行迁移或内容替换流程。

### Q: 为什么我开了插件，但新图还是本地地址？

A: 通常是以下原因：

- `replace_url` 没开启
- 云端配置不完整
- 插件未真正启用
- 你当前看的是缓存页面

## 简短结论

- 新图片：会自动上传到你配置的云储存。
- 前台地址：由 `replace_url` 决定是否自动切到云端。
- 旧图片：不会自动改，需要迁移或重写内容。
