# 媒体库 UI 增强功能说明

## 功能概述

媒体库 UI 增强功能为 WordPress 媒体库添加了丰富的云端状态显示和操作功能，让你可以方便地管理云端存储的附件。

## 核心功能

### 1. 云端状态列显示

在媒体列表中新增「云端状态」列，显示每个附件的云端存储状态：

#### 状态类型

**✅ 已上传**
- 显示绿色勾选图标
- 显示云服务商名称
- 提供复制 URL 功能
- 提供查看云端文件链接

**❌ 上传失败**
- 显示红色错误图标
- 显示错误信息（最多 30 字符）
- 提供重试上传功能

**○ 未上传**
- 显示灰色云图标
- 显示"未上传"状态
- 提供立即上传功能

### 2. 媒体编辑页面增强

在附件编辑页面添加「云端存储信息」元数据框：

#### 显示内容
- **上传状态**: 清晰显示当前状态
- **云服务商**: 显示使用的云存储服务商
- **文件 Key**: 显示云端文件 Key（可复制）
- **云端 URL**: 显示完整的云端 URL（只读）
- **上传时间**: 显示文件上传到云端的时间
- **缩略图信息**: 列出所有已上传的缩略图尺寸

#### 操作按钮
- **重新上传**: 重新上传文件到云端
- **复制 URL**: 一键复制云端 URL 到剪贴板

### 3. 媒体行操作扩展

在媒体列表的行操作中添加了云端相关操作：

**已上传的附件**:
- 查看云端文件（在新标签页打开）
- 复制 URL

**未上传的附件**:
- 上传到云端

### 4. 批量上传功能

支持批量选择附件并上传到云端：

#### 使用步骤
1. 进入媒体库页面
2. 勾选多个附件
3. 在「批量操作」下拉菜单中选择「上传到云端」
4. 点击「应用」
5. 等待上传完成

#### 批量操作结果
- 显示成功上传的数量
- 显示失败的数量
- 失败的附件会保留错误信息

### 5. 云端状态筛选

在媒体库顶部添加了云端状态筛选器：

**筛选选项**:
- **所有云端状态**: 显示所有附件
- **已上传**: 只显示已上传到云端的附件
- **未上传**: 只显示未上传的附件
- **上传失败**: 只显示上传失败的附件

### 6. 快速复制 URL

提供多种方式快速复制云端 URL：
- 在媒体列表中点击「复制 URL」
- 在媒体编辑页面点击「复制 URL」按钮
- 在行操作菜单中选择「复制 URL」

## 使用场景

### 场景 1: 检查附件上传状态

```
进入媒体库 → 查看云端状态列 → 识别需要上传的附件
```

### 场景 2: 单个附件上传

```
进入媒体库 → 点击「立即上传」/「重试上传」 → 等待上传完成 → 自动刷新
```

### 场景 3: 批量上传附件

```
进入媒体库 → 勾选多个附件 → 选择批量操作「上传到云端」 → 等待批量完成
```

### 场景 4: 查看附件详情

```
进入媒体库 → 点击附件标题 → 查看右侧「云端存储信息」元数据框
```

### 场景 5: 复制云端 URL

```
进入媒体库 → 点击「复制 URL」链接 → URL 已复制到剪贴板
```

## 技术实现

### 核心类：`WPMCS_Media_Library_Enhancer`

**主要方法**:
- `add_media_columns()`: 添加自定义列
- `render_media_column()`: 渲染列内容
- `add_row_actions()`: 添加行操作
- `add_meta_box()`: 添加元数据框
- `ajax_reupload_attachment()`: AJAX 重新上传
- `ajax_copy_cloud_url()`: AJAX 复制 URL
- `add_bulk_actions()`: 添加批量操作
- `handle_bulk_reupload()`: 处理批量上传
- `add_filter_dropdown()`: 添加筛选器
- `handle_filter()`: 处理筛选

### JavaScript 功能

**文件**: `assets/js/media-library.js`

**功能**:
- 重新上传处理
- 复制 URL 处理（支持现代浏览器和传统方法）
- 批量操作结果通知
- 通知动画效果
- 错误处理

### CSS 样式

**文件**: `assets/css/media-library.css`

**特性**:
- 响应式设计
- 状态徽章样式
- 工具提示
- 进度条动画
- 通知动画

## AJAX 接口

### 重新上传附件

**端点**: `wp_ajax_wpmcs_reupload_attachment`

**请求参数**:
```
action: wpmcs_reupload_attachment
attachment_id: 附件 ID
nonce: 安全令牌
```

**响应格式**:
```json
{
  "success": true,
  "data": {
    "message": "上传成功",
    "cloud_url": "https://cdn.example.com/file.jpg",
    "cloud_key": "uploads/file.jpg"
  }
}
```

### 复制云端 URL

**端点**: `wp_ajax_wpmcs_copy_cloud_url`

**请求参数**:
```
action: wpmcs_copy_cloud_url
url: 云端 URL
```

**响应格式**:
```json
{
  "success": true,
  "data": {
    "url": "https://cdn.example.com/file.jpg"
  }
}
```

## 权限控制

所有操作都需要相应的 WordPress 权限：
- **查看状态**: 无需特殊权限
- **上传/重新上传**: 需要 `upload_files` 权限
- **复制 URL**: 需要 `upload_files` 权限
- **批量操作**: 需要 `upload_files` 权限

## 性能考虑

### 数据库优化
- 使用 `postmeta` 表存储云端元数据
- 筛选查询使用 `EXISTS` 子查询提高性能
- 批量操作避免重复查询

### 前端优化
- 异步 AJAX 请求，不阻塞页面
- 轻量级 JavaScript，无外部依赖
- CSS 动画使用硬件加速

### 缓存建议
- 可以考虑添加对象缓存减少数据库查询
- 静态资源使用浏览器缓存

## 注意事项

### 1. 批量上传限制
- 一次建议不超过 50 个附件
- 大文件上传可能需要较长时间
- 建议分批次处理

### 2. 权限问题
- 确保云存储密钥有足够的权限
- 检查文件权限设置

### 3. 网络问题
- 上传失败会保存错误信息
- 可以通过「重试上传」重新上传
- 批量操作中的失败不会影响其他附件

### 4. URL 复制兼容性
- 优先使用现代 Clipboard API
- 降级到 `execCommand` 方法
- 浏览器不支持时会提示

## 故障排除

### 问题 1: 状态列不显示
**解决方案**:
- 确保插件已激活
- 检查是否有其他插件冲突
- 清除浏览器缓存

### 问题 2: 上传按钮无响应
**解决方案**:
- 检查浏览器控制台是否有 JavaScript 错误
- 确认 AJAX URL 正确
- 检查权限设置

### 问题 3: 批量上传失败
**解决方案**:
- 检查网络连接
- 验证云存储配置
- 查看错误日志

### 问题 4: URL 复制不工作
**解决方案**:
- 检查浏览器是否支持复制功能
- 尝试使用不同的浏览器
- 手动复制 URL
